## 2. PostgreSQL development packages on Ubuntu 24.04

For building C extensions against a single PG major:
- `postgresql-server-dev-XX` (e.g., `postgresql-server-dev-16`, `-17`, `-18` from PGDG): provides server headers (`include/server/...`), `pgxs.mk`, and `Makefile.global` per major version.
- `libpq-dev`: client libpq headers (needed if extension uses libpq, e.g., bgworker dial-back, FDWs).
- `postgresql-server-dev-all` (postgresql-common): provides `pg_buildext` and `dh_make_pgxs` for multi-version Debian builds. Pulls per-version `-server-dev-XX` as needed.
- Build essentials: `build-essential`, `pkg-config`, `bison`, `flex`, `libicu-dev`, `libssl-dev`, `libreadline-dev`, `zlib1g-dev`, `liblz4-dev`, `libzstd-dev`, `libxml2-dev`. For LLVM-JIT-aware extensions: `llvm-18-dev`, `clang-18`.
- PGXS itself only requires: a working `pg_config` (from `postgresql-server-dev-XX`), `make`, a C compiler, and `perl` for some test harnesses.

## 3. PGXS (Extension Building Infrastructure)

- Standard tail of every Makefile:
  ```
  PG_CONFIG = pg_config
  PGXS := $(shell $(PG_CONFIG) --pgxs)
  include $(PGXS)
  ```
- **`MODULES`**: list of single-file shared libraries (one .c -> one .so). **`MODULE_big`**: a single multi-source shared library; list objects in `OBJS`. Use `MODULE_big` for any non-trivial extension (most memory-DBMS work).
- **`EXTENSION = name`** + **`DATA = name--1.0.sql ...`** drive the SQL/control install.
- **Install layout** under `pg_config --sharedir`:
  - `extension/name.control`
